# Bach Guitar Generator — End-to-End Project Plan (with Token Factorization)

## 1) North-Star Goal

Generate new Bach-inspired pieces that are actually playable on classical guitar.

**Output formats (pragmatic)**
* v1 guarantees: ASCII tab + MIDI + MusicXML.
* GPX/GP export is optional and tooling-dependent;  treat as a later milestone.

The system favors ergonomics and musical plausibility with fast preview→refine loops.

---

## 2) System Overview

### 2.1 Modules 

* **Composer (NoteLM):** decoder-only Transformer that emits a beat-aware event stream enriched with chromatic semitone intervals.
* **Guitar Tabber:** maps symbolic pitches to (string, fret) via constrained beam search; optional learned TabCostModel.
* **Renderers:** ASCII tab, MusicXML (GPX optional), MIDI.
* **(Optional) BarPlanner:** light encoder-decoder that predicts per-bar descriptors ("BarPlan") to shape form; NoteLM conditions on these via a small bridge.

### 2.2 Data Flow (high level)

```
Score (MusicXML/MIDI) → Eventization (REMI+ with intervals) → BPE ids
                                      ↘ Bar descriptors (BarPlan)
Composer (NoteLM) ⇄ Inpainting/Repair → Symbolic piece

Symbolic piece → Tabber → (STR,FRET) sequence → Renderers → Outputs
```

---

### 2.3 Composer vs Tabber boundary

* **NoteLM output (v1):** symbolic music only (BAR/POS/DUR/VOICE + `ABS_*`, `MEL_INT12`, and `HARM_*`).
* **Tabber output:** `(STR, FRET)` assignment is produced after composition by a separate Tabber via constrained search.
* `STR_*` / `FRET_*` tokens are Tab-domain and are not generated by NoteLM in v1.

---

### 2.4 Token Stream -> Canonical Score (UI bridge)

Frontend/DB uses a canonical event model (see `frontend-readme.md`). The bridge from tokens is:

* **Timing:** BAR/POS_* update the time cursor; `DUR_{ticks}` becomes `durTick` in the canonical event.
* **Pitch:** `ABS_VOICE_*` anchors + `MEL_INT12` reconstruct `pitchMidi`.
* **Voice mapping:** for generated output, `VOICE_v` maps 1:1 to `voiceId = v` in the canonical score (single guitar part). Imported scores can use part/voice indices; eventization maps those to `VOICE_v` tracks for training.
* **Measures:** BAR tokens define measure boundaries; `timeSigMap` + `tpq` determine `lengthTicks`. Each measure gets a stable `measure.id`.
* **Event IDs:** `eventId` is assigned during token-to-canonical conversion (UUID/monotonic) for UI tracking; new IDs are created only for regenerated events.

## 4) Notation & Tokenization

### 4.1 Ground Rules

* **Time grid:** ticks-per-quarter (`TPQ`) fixed at 24.

    * **Rationale:** TPQ=24 is the LCM of binary (8th, 16th, 32nd) and ternary (triplets) subdivisions. At TPQ=12, a 32nd note = 1.5 ticks (unrepresentable as integer), causing rhythmic rounding errors in Bach's ornate passages (Sarabandes, Partitas). TPQ=24 handles 8ths (12), 16ths (6), 32nds (3), triplet 8ths (8), and sextuplets without rounding.

* **Event sparsity:** `POS_t` tokens are emitted only when at least one voice event starts at that offset (pitched onset or explicit `REST_{ticks}` if enabled). Time between events is implicit.

* **Cross-bar continuity:** Training/inference operate on piece streams or multi-bar windows so per-voice state (`prev_pitch`, `active_until`) carries across barlines.

    * If strict "one row per bar" loading is used: any note crossing a barline must be split at the boundary (requires a tie/continuation convention).

* **Anchors (absolute pitch):**

    * `ABS_VOICE_{v}_{MIDI}` may appear at bar start if that track is sounding at the boundary (including ties), and/or before the first onset of that track within a window if `prev_pitch[v]` is undefined.

    * `ABS_VOICE_{v}_{MIDI}` may also be emitted on large leaps and long silences (policy below).

    * (Optional conditioning-only anchors) `ABS_LOW_{MIDI}` and `ABS_HIGH_{MIDI}` may appear at bar start to expose the global low/high boundary pitches. These do not set per-track state.

* **Intervals (chromatic):**

    * **Melodic:** `MEL_INT12 ∈ [-24..+24]` — semitone delta vs previous note in the same voice.

    * **Harmonic reference (causal, robust):** `ref_pitch_t` is defined as the lowest active pitch at time `t` (considering onsets and sustaining notes), independent of `VOICE` index. This avoids reference-jump artifacts caused by track crossings, rests, or remapping.

        * **Negative octaves:** permitted (voice crossings relative to the reference).

        * **`HARM_*` tokens are emitted only for pitched onsets.**

    * **Optional diatonic steps:** `MEL_STEP7`, `HARM_STEP7` are aux/side features only.

* **Versioning + stable merges:** train BPE/Unigram once on the serialized stream; freeze merges.

### 4.1.1 Polyphony, Tracks, and Doubling Collapse (IMPORTANT)

**Token voices are tracks (not SATB parts).**
`VOICE_v` is an internal *track index* used to preserve melodic continuity across time (including voice crossings).
It is not guaranteed to correspond to score "parts" (Violin I, Alto, etc.) or to a fixed register rank (lowest/highest).

**Polyphony cap (MAX_VOICES = K, default K=8).**
The Composer is trained with a fixed cap of `K` simultaneous logical tracks (`VOICE_0..VOICE_{K-1}`) for batching and model stability.
Scores may contain more than `K` notated parts; those are handled by deterministic collapsing/reduction before tokenization.

**Orchestration multiplicity is intentionally discarded.**
We intentionally discard orchestration multiplicity by collapsing exact pitch doublings (after converting to sounding pitch).
The representation is invertible to a "collapsed score," not to the original instrumentation.

**Pre-tokenization normalization pipeline (canonical):**
0) Convert to sounding pitch (concert pitch) so doublings match by actual pitch (important for transposing instruments).
1) Normalize ties into single sustained notes (merge tied segments across barlines).
2) If measure bookkeeping is inconsistent (over/underfull), rebar and split/tie across barlines.
3) Quantize offsets/durations to the fixed TPQ grid.
   - Sanitize invalid tuplets like `<normal-notes>3/2</normal-notes>` by expanding to an integer ratio before parsing.
4) Collapse pitch-class doublings at each onset:
   - If multiple notes start at the same tick `t` with the same pitch class (unison or octave), keep up to 3 distinct octaves.
   - If exact pitch duplicates occur, keep the max duration (or union by extending to the latest end time).
   - (Optional) record a token `DUP_{n}` indicating how many notes were collapsed into that event.
5) Track assignment (continuity):
   - Assign remaining note events to `VOICE_v` tracks by continuity (minimize |delta_pitch| vs each track's last pitch and respect `active_until`).
   - Crossings are allowed; a track can move above/below another over time.

### 4.2 Event Schema

* **Structural:** `BAR`, `TIME_SIG_*`, `KEY_*`, `TEMPO_*`, `POS_*`

* **Voice events (sparse, duration-first):** `VOICE_{0..K-1}`, `DUR_{ticks}`, `REST_{ticks}` (optional)

    * `DUR_{ticks}`: required for every pitched onset; sustain is implied by duration.

    * `REST_{ticks}`: optional explicit silence spans; omit if onsets-only is sufficient.

* **Anchors:** `ABS_VOICE_{v}_{MIDI}` (per-track), optional `ABS_LOW_{MIDI}`, `ABS_HIGH_{MIDI}`

* **Doubling (optional):** `DUP_{1..N}` emitted only when exact unison doublings were collapsed at that onset.
  If omitted, doubling collapse still happens; the multiplicity signal is simply not modeled.

* **Intervals (pitched onsets only):**

    * `MEL_INT12_{-24..+24}`
    * `HARM_OCT_{-2..4 | NA}`
    * `HARM_CLASS_{0..11 | NA}`

* **Guitar (Tabber domain):** `STR_{1..6}`, `FRET_{0..19}` (not generated by NoteLM v1)

* **Harmony/dynamics (optional):** `CHORD_*`, `VEL_*`

* **Auxiliary (non-serialized):** `NEXT_MEL_INT12_*` teacher-forced targets.

**Spec** (`src/tokens/schema.py`)

```python
from dataclasses import dataclass
from typing import Dict, List

@dataclass
class EventSpec:
    version: str  # "remi_tab_v1"
    tpq: int      # 24
    tokens: Dict[str, List]

@dataclass
class DescriptorSpec:
    fields: List[str] = ("TIME_SIG","KEY","CHORD_FN","DENSITY","CADENCE","DIFFICULTY")
```

DescriptorSpec fields correspond 1:1 with the BarPlan attributes referenced later in the dataset builder.

### 4.3 Interval Math (deterministic)

**Harmonic reference definition (deterministic, causal)**

At each emitted `POS_t`:

1. A chord slice contains all voice events after `POS_t` until the next `POS_*` or `BAR`.

2. Within a slice, order voice events deterministically by `VOICE_0..VOICE_{K-1}`.

3. Maintain per-voice state:

    * Voice `v` is active at time `t` if `active_until[v] > t` (sustaining) or it has an onset at `POS_t`.

    * If `active_until[v] <= t`, the voice is not sustaining; its pitch at time t is undefined unless it has an onset at `POS_t`.

4. Compute reference:

    * Let `active_pitches_t` be the set of pitches that are active at time `t`
      (either sustaining via `active_until` or onsets at `POS_t`).
    * `ref_pitch_t = min(active_pitches_t)` if non-empty; otherwise no reference exists.

**Vertical intervals (pitched onsets only)**

* If `ref_pitch_t` exists:

    * `diff = pitch - ref_pitch_t`
    * `(octave, harm_class) = divmod(diff, 12)`
    * emit `HARM_OCT_{octave}`, `HARM_CLASS_{harm_class}`

* Else (rare — e.g., before first onset):

    * emit `HARM_OCT_NA`, `HARM_CLASS_NA` (and optionally down-weight/skip HARM loss)

**Melodic intervals**

* `MEL_INT12 = curr_pitch - prev_pitch[v]`

### 4.4 Pitch Reconstruction (source of truth)

Each voice maintains `prev_pitch[v]` and `active_until[v]`.

1. **Anchors:** `ABS_VOICE_{v}_{MIDI}` sets `prev_pitch[v] = midi`.
   (Optional `ABS_LOW_{MIDI}` / `ABS_HIGH_{MIDI}` do not set per-track state.)

2. **Pitched onset:** `VOICE_v, DUR_d, MEL_INT12_Δ, ...`

    * **Invariant:** A pitched onset for `VOICE_v` must have a defined `prev_pitch[v]` earlier in the stream/window
      (via `ABS_VOICE_{v}_{MIDI}`); otherwise, repair inserts an anchor deterministically.
    * `pitch = prev_pitch[v] + Δ`
    * `prev_pitch[v] = pitch`
    * `active_until[v] = t + d`

3. **Explicit rest (optional):** `VOICE_v, REST_r` marks voice inactive until `t + r`; `prev_pitch[v]` remains unchanged.

**Truth:** absolute pitch is determined exclusively by `ABS_VOICE_* + MEL_INT12`. `HARM_*` are derived features and must match `pitch - ref_pitch_t` when reference exists.

### 4.5 Voice event encoding

Within each `POS_t` slice, for each voice that has an event at time t:

**A) Pitched onset**

```
VOICE_v, DUR_{ticks}, (optional) DUP_{n}, MEL_INT12_{Δ}, HARM_OCT_{o|NA}, HARM_CLASS_{c|NA}
```

* **For v1, every pitched onset emits exactly one `HARM_OCT_*` and one `HARM_CLASS_*` (either concrete or NA).**
* If a reference exists at `POS_t`, `{o,c}` must be present and consistent.
* If no reference exists, emit `{NA,NA}`.

**B) Optional explicit rest**

```
VOICE_v, REST_{ticks}
```

**Parsing rule:** After `VOICE_v`, if the next token is `REST_{ticks}`, the event ends; otherwise it must be `DUR_{ticks}` followed by optional `DUP_{n}`, `MEL_INT12_*`, `HARM_OCT_*`, `HARM_CLASS_*`.

### 4.6 Consistency + debug modes

**v1 pitch truth:** `ABS_VOICE_* + MEL_INT12`.

**Dataset invariant:**

* reconstruct pitch from anchors + `MEL_INT12`,
* compute `ref_pitch_t` causally from (`active_until`, onsets at t),
* verify emitted `HARM_*` matches `(pitch - ref_pitch_t)` when reference exists.

**Inference:**

* sample `MEL_INT12` (determines pitch),
* compute `ref_pitch_t`,
* either force `HARM_*` (constrained) or sample + log mismatches (debug).

### 4.7 Anchor insertion policy

* **Bar start:** emit `ABS_VOICE_{v}_{MIDI}` for any track sounding at the boundary (including ties).

* **Optional global anchors:** emit `ABS_LOW_{MIDI}` / `ABS_HIGH_{MIDI}` at bar start to expose the global low/high boundary pitches.

* **Voice re-entry:** if silence > N ticks (e.g., 48), emit `ABS_VOICE_{v}_{MIDI}`.

* **Large leaps:** if `abs(Δ) > MEL_RANGE`, emit `ABS_VOICE_{v}_{MIDI}`.

* **Drift prevention:** optional periodic anchors every N bars.

### 4.8 Interval ranges (practical policy)

* Default melodic range starts as `MEL_INT12 ∈ [-24, +24]`.
* Default harmonic octave range is `HARM_OCT ∈ [-2, 4]`.
* If either range causes frequent repairs or anchor insertions, we widen the range (preferred) rather than silently clamping.
* Example: a high line above a low reference may exceed octave +4 in rare cases.
* We track histogram tails and repair counts in `stats.json` to decide when to widen.

### 4.9 BPE (Stage-2, optional)

* BPE (Byte-Pair Encoding) optionally merges frequent token sequences to reduce sequence length and improve context efficiency.
* Safety rule: BPE merges must not cross structural boundaries (`BAR`, `POS_*`, or note-bundle boundaries at `VOICE_*`).
* Allowed: merges inside a voice bundle (e.g., `DUR_* + MEL_INT12_*`, `HARM_OCT_* + HARM_CLASS_*`, `VOICE_* + REST_{ticks}`).
* Stage-1 works without BPE; Stage-2 adds it as an optimization.

### 4.10 Invertibility + Canonical ordering

**Invertibility:** Bar-start anchors (when the track is sounding) + voice-wise `MEL_INT12` continuity (with `ABS_VOICE_*` on re-entry/large leaps) reconstruct absolute pitch exactly (enharmonic spelling aside). When reference pitch is defined, `pitch - ref_pitch_t` maps uniquely to `HARM_OCT/HARM_CLASS`.

**Note on orchestration:** because exact unison doublings are collapsed, round-trip reconstruction targets the
collapsed score representation (unique pitch events), not the original instrumentation/part counts.

**Canonical ordering:** Composer stream order for pitched onsets: `VOICE_*`, `DUR_*`, optional `DUP_*`, `MEL_INT12_*`, `HARM_OCT_*`, `HARM_CLASS_*`. For explicit rests: `VOICE_*`, `REST_{ticks}`. The tabbed stream appends `STR_*`, `FRET_*` after `HARM_CLASS_*` (for pitched onsets only).

**Example A: NoteLM output (composer stream; `HARM_*` derived in constrained mode)** (from middle of piece, bar N, showing different voice event types)

```
BAR, TIME_SIG_4_4, KEY_Em, POS_0,
ABS_VOICE_0_40, ABS_VOICE_3_76,
VOICE_0, DUR_6, MEL_INT12_0,  HARM_OCT_0, HARM_CLASS_0,  # E2 - pitched onset (track 0)
VOICE_1, DUR_2, MEL_INT12_+7, HARM_OCT_1, HARM_CLASS_7,  # B3 - pitched onset
VOICE_2, DUR_4, MEL_INT12_+4, HARM_OCT_2, HARM_CLASS_11, # Eb4 - pitched onset
VOICE_3, REST_3,                                         # track 3 silent for 3 ticks
POS_3,
VOICE_1, DUR_2, MEL_INT12_+2, HARM_OCT_1, HARM_CLASS_9,  # C#4 - new onset
VOICE_3, DUR_2, MEL_INT12_+12, HARM_OCT_3, HARM_CLASS_4, # G4 - track 3 re-enters
...
```

**Example B: after Tabber (adds STR/FRET to pitched onsets only)**

```
BAR, TIME_SIG_4_4, KEY_Em, POS_0,
ABS_VOICE_0_40, ABS_VOICE_3_76,
VOICE_0, DUR_6, MEL_INT12_0,  HARM_OCT_0, HARM_CLASS_0,  STR_6, FRET_0,  # pitched onset gets tab
VOICE_1, DUR_2, MEL_INT12_+7, HARM_OCT_1, HARM_CLASS_7,  STR_4, FRET_2,  # pitched onset gets tab
VOICE_2, DUR_4, MEL_INT12_+4, HARM_OCT_2, HARM_CLASS_11, STR_2, FRET_0,  # pitched onset gets tab
VOICE_3, REST_3,                                                         # no tab for REST
POS_3,
VOICE_1, DUR_2, MEL_INT12_+2, HARM_OCT_1, HARM_CLASS_9,  STR_3, FRET_4,  # pitched onset gets tab
VOICE_3, DUR_2, MEL_INT12_+12, HARM_OCT_3, HARM_CLASS_4, STR_1, FRET_3,  # pitched onset gets tab
...
```

**Example C: Bar 0 (Piece Start)**

```
BAR, TIME_SIG_4_4, KEY_C, POS_0,
ABS_VOICE_0_36, ABS_VOICE_3_60,
VOICE_0, DUR_12, MEL_INT12_0, HARM_OCT_0, HARM_CLASS_0,  # pitch = 36 + 0 = 36 (C2)
VOICE_3, DUR_12, MEL_INT12_0, HARM_OCT_2, HARM_CLASS_0,  # pitch = 60 + 0 = 60 (C4)
...
```

`MEL_INT12_*` tokens always include explicit signs (`+`, `-`, or none for zero) to keep directionality obvious. At the very first onset (Bar 0), `MEL_INT12_0` means "0 semitones from the anchor".

`NEXT_MEL_INT12_*` denotes teacher-forced look-ahead deltas computed offline for metrics/aux-loss; they are never serialized into the autoregressive stream.

This octave/class factorization keeps "what note" (`HARM_CLASS`) separate from "how far above the reference pitch" (`HARM_OCT`), which mirrors classical-guitar ergonomics (string choice, hand position) and removes the cognitive overhead of wrapped signed integers.

## 5) Dataset & Processing

### 5.1 Sources

* **Composer:** JSB chorales, inventions, preludes/fugues (PD/CC).
* **Guitar (for tab pretrain/cost later):** Bach lute suites (BWV 995–1000), Sor, Carcassi, etc. Prefer MusicXML; GPX/GP optional.

### 5.2 Builder (`scripts/make_dataset.py`)

1. Parse MusicXML -> convert to sounding pitch
2. Normalize ties (merge tied segments into single sustained notes)
3. Quantize offsets/durations to TPQ=24 ticks
4. Collapse exact unison doublings at each onset (same tick + same MIDI pitch)
   - keep one; duration = max (or union to latest end)
   - (optional) record `DUP_{n}`
5. Assign events to VOICE tracks by continuity (minimize |delta_pitch| using prev_pitch/active_until; crossings allowed)
   - Implementation tip (scripts/make_dataset.py): a simple + robust hybrid:
     - If len(new_notes) <= 1 or len(active_tracks) <= 1: greedy.
     - Else: Hungarian (linear sum assignment) as the default.
6. Compute harmonic reference pitch per POS_t as the lowest active pitch (onsets + sustains)
7. Pack events -> BPE ids
8. Label BarPlan per bar -> shard + stats

### 5.3 Invariant handling (QA vs production)

* **QA mode:** invariant violations raise (used in unit tests).
* **Dataset/production mode:** violations are repaired deterministically and logged (counts reported in `stats.json`).

Repairs include:

* insert an `ABS_VOICE_{v}_{MIDI}` anchor on voice re-entry or large leaps (if enabled),
* widen interval ranges if clamping causes frequent repairs (see "Interval ranges" below).

**Artifacts**

* `data/processed/events.parquet`: `[piece_id, bar, token_ids, bar_index]`
* `data/processed/barplans.parquet`: `[piece_id, bar, BarPlan(json)]`
* `data/processed/stats.json`: token counts; interval histograms; repair counts; key balance; avg bar length

**Quality checks**

* Round-trip decode of random bars ≈ exact pitch match
* Interval histograms: MEL dominated by stepwise; HARM_CLASS peaks at 3rds/6ths

## 6) Models

### 6.1 Composer — NoteLM

* **Backbone:** decoder-only Transformer (8–12 layers, d≈512–640, heads≈8, context 2048–3072). Rotary/relative attention.
* **Inputs:** BPE ids from the event stream (with anchors + intervals).
* **Controls:** `{key, style(Chorale|Fugue|Prelude), difficulty, measures}` as prefix tokens.
* **Aux heads (optional):** per-bar key, cadence, and/or `NEXT_MEL_INT12`.
* **API**

```python
class NoteLM(nn.Module):
    def forward(self, ids, attn_mask, desc_embed=None):
        ...
```

* **Regularization:** label smoothing, token dropout on `HARM_CLASS`, repetition penalty at sampling.
* **Training schedule:** pretrain NoteLM on symbolic corpora (no STR/FRET) → fine-tune on Bach SATB events → pretrain TabCostModel separately on tabs; a later v2 can explore co-generation of tab tokens.

### 6.2 (Optional) Bar-level Planner — BarPlanner

* **Goal:** emit BarPlan\[1..N] to improve global form (cadences, density, modulations).
* **Model:** tiny encoder-decoder or AR decoder over plan tokens.
* **Conditioning bridge:** MLP maps each BarPlan → `desc_embed[b]` injected into NoteLM at each BAR (no weight surgery).

**Conditioning bridge implementation:**

```python
class NoteLM(nn.Module):
    def forward(self, ids, attn_mask, desc_embed=None):
        # ids: (batch, seq_len)
        # desc_embed: (batch, num_bars, desc_dim) or None

        token_emb = self.token_embedding(ids)  # (batch, seq_len, d_model)

        if desc_embed is not None:
            # Inject desc_embed at BAR positions
            # bar_positions: boolean mask of BAR token locations
            bar_positions = (ids == self.bar_token_id)
            # Project desc_embed to d_model and add to token_emb at bar_positions
            # Implementation: for each BAR token at position i, add desc_embed[bar_idx]

        return self.transformer(token_emb, attn_mask)
```

**Note:** v1 skips BarPlanner; defer to v2.

## 7) Decoding, Rule Guidance & Repair

### 7.1 Sampler

* Top-p / Temperature with anti-repetition n-gram penalty.
* `Γ` (density): nudges chord/position change rate at bar scope.
* `α` (strictness): SCG wrapper that rescores candidates using rules (voice-leading, range, parallel 5ths/8ves, rhythm sanity).

### 7.2 Interval-aware soft filters

* Penalize inner-voice leaps `|MEL_INT12|>7` unless cadential.
* Gentle bias toward consonant `HARM_CLASS` (3rds/6ths) in chorale mode; octave tokens steer voicing spread separately.

### 7.3 Blocked-Gibbs Repair

* Bar-level resampling over masked bars/voices using the same rules and interval penalties.
* Accept when: no hard violations, parallels below threshold, and score improves.

## 8) Guitar Tabber

### 8.1 Candidate Enumeration

* For each onset, list legal `(string,fret)` per pitch under tuning/capo; include state (hand position, held fingers).

### 8.2 Search (Heuristic Beam)

* Beam over time with width 8–32; prune by near-best cost and candidate caps.
* **Hard constraints:** max span per slice, max shift per beat, string-order in chords, barre duration caps, no impossible crossings.
* **Stability:** “zero-join” bonus when maintaining positions; chorale mode prefers open basses.

Tabbing runs in bar windows (e.g., 4-8 bars) with overlap and cached hand-state to keep preview-refine loops responsive. A Viterbi / shortest-path formulation over time-slices is a compatible alternative to beam search.

### 8.3 Learned TabCostModel (optional)

* **Inputs:** candidate + previous state + bar/beat features + chord-shape stats + voice→string consistency.
* **Network:** small FFN/LSTM; trained on human tabs + synthetic near-misses; optional RL fine-tune from beam rollouts.

### 8.4 Difficulty Tiers

Beginner / Intermediate / Advanced mapped to (span, shift, barre) ceilings; enforced in constraints and exposed in UI.

## 14) Testing & Gotchas

* **Round-trip decoding:** random bars reconstruct original MIDI (±1 tick tolerance near tie joins).
* **Voice re-entries:** ensure MEL uses last sounding pitch per voice (`ABS_VOICE_*` restores absolute pitch).
* **Ties/durations:** active-pitch sets include sustained notes derived from durations for correct reference-pitch detection.
* **Transposing instruments:** always convert to sounding pitch before collapsing doublings, or doublings will not match.
* **Doubling collapse:** unison collapse intentionally discards "how many instruments played it"; only pitch/time/duration remain.
* **Silent slices:** no note events emitted; only `REST_{ticks}` if you need explicit silence.
* **Edge intervals:** unit tests for out-of-range handling (repairs/widening).

def get_harm_oct_class(pitch: int, ref_pitch: int, min_octave: int = -2, max_octave: int = 4):
    """
    Factorizes harmonic distance into octave + chromatic class.
    Allows negative octaves for voice crossings (pitch < ref_pitch).
    ref_pitch is the lowest active pitch at time t.
    In QA mode, out-of-range should raise; in production, log + repair or widen range.
    """
    diff = pitch - ref_pitch
    octv, klass = divmod(diff, 12)
    if octv < min_octave or octv > max_octave:
        raise ValueError(f"HARM_OCT out of range: {octv} not in [{min_octave}, {max_octave}]")
    return octv, klass

def clamp(v, lo, hi):
    return lo if v < lo else hi if v > hi else v
